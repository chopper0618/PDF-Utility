# ROADMAP

## v1.0 までの考え方

v1.0 は、PDF の読み込み、ページ単位の整理、結合・書き出しが安定して行える状態を目指す。

派手な機能よりも、まずは「安心してページ操作できること」「画面表示と実際の PDF 構成がズレないこと」を優先する。

## Sprint 0.7.0: Page Map 改善

完了。

- Page Map 同期
- Page Map クリックジャンプ
- ドラッグ中自動スクロール
- 指定ページ番号への直接移動
- Ribbon 表記を「移動...」に短縮

## Sprint 0.7.1: 移動ダイアログ改善

完了。

- ブラウザ標準 `prompt()` を独自モーダルへ置き換える
- ダイアログタイトルを「指定ページへ移動」にする
- 入力欄にページ番号を入れる
- Enter で実行、Esc でキャンセル
- 範囲外入力時はその場でエラー表示
- 背景クリックまたはキャンセルで閉じる
- 入力欄で Backspace が削除ショートカットに誤爆しない

## Sprint 0.7.2: 削除操作UX改善 / Page Map表示整理

完了。

- 削除後に Status Bar へ「Undoで元に戻せます。」を表示
- Delete / Backspace 削除は即時実行のまま維持
- Page Map 下部のファイル一覧を削除
- Page Map のファイル名を先頭＋末尾の中間省略表示に変更
- マウスオーバーで全体情報を確認できる状態を維持

## Sprint 0.7.3: サムネールファイル名表示改善

候補。

## v1.0 の完了条件

- PDF を読み込める
- ページを削除できる
- ページを並び替えできる
- 複数 PDF を結合できる
- 編集後の PDF を書き出せる
- Page Viewer と Page Map の状態が一致する
- エラー時にユーザーが次に何をすればよいかわかる
- docs に仕様と現在地が残っている


## Sprint 0.7.4: サムネールカードレイアウト調整

サムネールカード下部のファイル名を最大2行表示にし、先頭と末尾の判別性を上げる。Page Map は1行表示を維持し、一覧性を優先する。
