# PROJECT_OVERVIEW

## プロジェクト名

PDF Utility

## 目的

PDF の結合・ページ並び替え・削除・移動を、ブラウザ上で直感的に行える Web アプリを作成する。

このアプリは、PDF を扱う作業で発生しやすい「ページ順の確認」「不要ページの削除」「別 PDF との結合」「ページ単位の移動」を、できるだけ迷わず、画面上の見た目と操作結果が一致する形で行えることを目指す。

## 想定ユーザー体験

1. PDF ファイルを読み込む
2. 読み込み中は進捗が表示される
3. ページ一覧を確認する
4. ページをドラッグ、削除、移動コマンドで整理する
5. 左側の Page Map で全体構成を確認する
6. 必要に応じて Page Map から該当ページへジャンプする
7. 結合または編集後の PDF を書き出す

## 重視すること

- 「今どのページを操作しているか」がわかること
- 大量ページでも移動しやすいこと
- 表示領域外のページへも自然に移動できること
- 左側の Page Map と実際のページ順が常に一致すること
- 読み込み状態、ページ数、操作結果がユーザーに明示されること
- 小さな Sprint 単位で安全に改善すること

## 現時点での開発方針

PDF Utility は、いきなり大規模な機能追加をするのではなく、実際に使うときの「ひっかかり」を Sprint 単位で解消していく。

直近では、PDF 読み込み中のプログレス表示と、ステータスバーの「〇ページ読み込み完了」表示は実装済みとして扱う。最近使ったファイル機能は現時点では不要。

## 次に改善する領域

次の Sprint 0.7.0 では、Page Map とページ移動の操作性を改善する。

対象は以下。

- ドラッグ中の自動スクロール
- 指定ページ番号への直接移動
- 並び替え・削除・移動後の Page Map 同期
- Page Map クリックによるページジャンプ

## 成功条件

大量ページ PDF を扱っても、ユーザーが「今どこにいて、どのページをどこへ移動したか」を見失わないこと。
