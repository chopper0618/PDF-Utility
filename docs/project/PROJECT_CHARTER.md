# PROJECT_CHARTER

## 開発理念

PDF Utility は、PDF ページ操作を「見たまま、迷わず、安全に」行えることを最優先にする。

高度な機能を増やす前に、日常的に使う操作を快適にする。

## 守るべき原則

### 1. 画面と内部状態を一致させる

Page Viewer、Page Map、Status Bar、内部の Page Order は常に同じ状態を指す。

### 2. ユーザーを迷わせない

読み込み中、操作中、完了後に、アプリが何をしているかを表示する。

### 3. 大量ページでも破綻しない

少数ページでは使えても、大量ページで移動しづらい UI は改善対象とする。

### 4. 小さく作って確認する

一度に大きく変更しない。Sprint と Issue に分けて、安全に改善する。

### 5. docs を残す

チャットが重くなっても、新しいチャットで開発を継続できるよう、判断・用語・仕様を docs に残す。

## やらないことの判断

現時点では「最近使ったファイル」は不要。操作性の中核である Page Map とページ移動を優先する。

## 品質の基準

- 操作結果が目で確認できる
- Undo が未実装でも、操作前後の状態がわかりやすい
- エラー時に原因と次の行動がわかる
- コードの状態管理が追いやすい
- docs と実装が矛盾しない
