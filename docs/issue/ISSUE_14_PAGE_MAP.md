# ISSUE_14_PAGE_MAP

## Issue 名

Issue #14「ページ移動・Page Map改善」

## 背景

現在、PDF Utility ではファイル操作をより快適にする必要がある。

特に、ページを移動するときに表示領域外のページへ移動しにくいこと、移動コマンドが「何ページ分動かすか」という指定になっていて大量ページでは直感的でないこと、左側の Page Map に並び替え後の状態が反映される必要があることが課題。

## 目的

大量ページ PDF でも、ページ移動・並び替え・全体確認を自然に行えるようにする。

## 対象範囲

### 対象

- Page Map 同期
- Page Map クリックジャンプ
- ドラッグ中自動スクロール
- 指定ページ番号への直接移動
- 操作後の Status Bar 更新

### 対象外

- 最近使ったファイル
- PDF Engine の大幅変更
- 高度な履歴管理
- 独自モーダル UI

## 要件

### 1. Page Map 同期

並び替え・削除・移動コマンド後、Page Map は現在の Page Order を反映する。

完了条件。

- ドラッグ並び替え後に Page Map の順番が変わる
- 削除後に Page Map から削除ページが消える
- 移動コマンド後に Page Map の順番が変わる

### 2. Page Map クリックジャンプ

Page Map の項目をクリックすると、Page Viewer の該当ページへスクロールする。

完了条件。

- クリックしたページが Page Viewer で表示される
- 選択状態が Page Viewer と Page Map で一致する
- Status Bar に移動結果が表示される

### 3. ドラッグ中自動スクロール

Page Viewer 内でページをドラッグ中、表示領域の上端または下端に近づくと自動スクロールする。

完了条件。

- 上方向へスクロールできる
- 下方向へスクロールできる
- ドロップ後の Page Order が正しい
- スクロールが暴走しない

### 4. 指定ページ番号への直接移動

選択ページを指定したページ番号へ直接移動できる。

方針。

- 「上へ移動」「下へ移動」は廃止する
- Ribbon には「指定ページへ」「先頭へ」「末尾へ」を置く
- 単一ページ選択時は、そのページを指定ページ番号へ移動する
- 複数ページ選択時は、選択ページ群をまとめて移動し、指定ページ番号を移動後の先頭位置として扱う
- 入力値が 1 未満、総ページ数超過、数値以外の場合は移動しない

完了条件。

- 任意のページを指定ページ番号へ移動できる
- 複数選択ページをまとまりとして移動できる
- 先頭へ移動できる
- 末尾へ移動できる
- 不正な入力でページ順が変わらない
- 移動後に Page Viewer と Page Map が同期する

## 推奨実装順

1. Page Order 更新後に Page Viewer / Page Map を再描画する共通関係を確認する
2. Page Map 同期を修正する
3. Page Map クリックジャンプを追加する
4. 指定ページ番号への直接移動を追加する
5. ドラッグ中自動スクロールを追加する
6. Status Bar 表示を整える

## テストケース

- 1つの PDF、10ページ
- 1つの PDF、50ページ以上
- 複数 PDF を読み込んだ状態
- 中央ページを3ページ目へ移動
- 複数選択ページを3ページ目へ移動
- 先頭ページを1ページ目へ移動しても破綻しない
- 末尾ページを最終ページへ移動しても破綻しない
- 0、空欄、文字列、総ページ数超過を入力してもページ順が変わらない
- Page Map クリックで先頭、中央、末尾へ移動
- ドラッグで表示外の位置へ移動
