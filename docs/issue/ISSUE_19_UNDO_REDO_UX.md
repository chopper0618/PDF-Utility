# Issue #19: Undo / Redo UX refinement

## Sprint

Sprint 0.7.5

## Goal

Undo / Redo を「動く機能」から「安心して使える操作履歴」に引き上げる。

PDF Utility では、削除・移動・並び替え・回転・複製が日常的に発生する。
そのため、操作後に戻せること、戻した結果、やり直せることが明確に伝わる必要がある。

## Scope

- Undo 実行時に、何を元に戻したかをステータスバーへ表示する
- Redo 実行時に、何をやり直したかをステータスバーへ表示する
- Undo 後は Redo できることを文言で示す
- Redo 後は Undo できることを文言で示す
- 入力欄・モーダル中のショートカット誤爆防止は維持する

## Out of scope

- 操作履歴一覧パネル
- 確認ダイアログの追加
- 履歴の永続化
- 複数段階履歴の可視化UI

## UX policy

確認ダイアログを増やしてクリック数を増やすのではなく、
削除・移動などの操作は即時実行し、Undo / Redo で安心して戻せる設計を優先する。

## Acceptance criteria

- 削除後に Undo すると「削除を元に戻しました。Redoでやり直せます。」と表示される
- 削除 Undo 後に Redo すると「削除をやり直しました。Undoで元に戻せます。」と表示される
- 移動・並び替え・回転・複製でも同様に操作名が表示される
- Undo / Redo 後も Page Map とサムネールが同期する
- 入力欄・モーダル中に Backspace / Delete がページ削除へ誤爆しない
