# Issue #22: Page Map drag reorder

## 目的

Page Map を「見るだけの一覧」から、ページ構成を直接操作できる地図へ進める。

中央の Page Viewer だけでなく、左側の Page Map でもページをドラッグして並び替えられるようにする。

## 背景

大量ページ PDF では、中央サムネールだけでページの全体構成を把握しながら並び替えるのが難しい。Page Map はページ順の俯瞰に向いているため、ここでもドラッグ並び替えできると操作距離を減らせる。

## 実装対象

- Page Map のページ項目をドラッグ可能にする
- ドラッグ位置に before / after の挿入インジケータを表示する
- Drop で `state.pages` の順番を更新する
- Page Viewer と Page Map の順番を同期する
- Undo / Redo に対応する
- Page Map クリックジャンプとドラッグ操作が競合しないようにする
- Page Map のファイル名末尾が切れにくいように表示を調整する

## やらないこと

- Page Map 上での削除、回転、複製ボタン追加
- Page Map 上の複雑なコンテキストメニュー
- Page Map の2行ファイル名表示

Page Map は一覧性を優先するため、操作を増やしすぎない。

## 受け入れ条件

- Page Map 上でページをドラッグして並び替えできる
- 並び替え後に Page Viewer 側も同じ順番になる
- 並び替え後に Page Map 側も同じ順番になる
- Undo / Redo で Page Map 並び替えを戻せる、やり直せる
- Page Map クリックジャンプが引き続き動く
- Page Map のファイル名末尾の識別情報が以前より残りやすい
