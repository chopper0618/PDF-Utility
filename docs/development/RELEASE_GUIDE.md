# RELEASE_GUIDE

## 目的

v1.0 に向けて、変更内容を安全にまとめるための手順。

## リリース前チェック

- 主要機能が動く
- Page Viewer と Page Map が同期する
- PDF 読み込み進捗が表示される
- 書き出しができる
- エラー時の表示がある
- docs が更新されている
- CHANGELOG が更新されている

## バージョン管理

Sprint 単位でバージョンを進める。

| Version | 内容 |
|---|---|
| 0.7.0 | Page Map 改善 |
| 0.8.0 | ページ操作 UX 安定化 |
| 0.9.0 | 書き出し・エラー処理確認 |
| 1.0.0 | 実用版 |

## CHANGELOG 更新

`release/CHANGELOG.md` に以下を記録する。

- 追加した機能
- 修正した不具合
- 変更した仕様
- 未解決の課題

## リリース判断

「使える」だけでなく、「次のチャットや後日の自分が状態を理解できる」こともリリース条件に含める。
