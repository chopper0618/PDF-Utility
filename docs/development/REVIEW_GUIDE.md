# REVIEW_GUIDE

## 目的

変更後に、機能が動くだけでなく、状態のズレや UX の違和感がないか確認する。

## レビュー観点

### 1. 状態管理

- Page Order が正しく更新されるか
- selectedPageId が削除済みページを指していないか
- Page Viewer と Page Map が同じ順番か

### 2. UI

- 選択中ページがわかるか
- Page Map の表示がわかりやすいか
- 移動後にユーザーが見失わないか
- Status Bar の文言が自然か

### 3. 操作性

- ドラッグ中に自動スクロールできるか
- 上端・下端でスクロールが暴走しないか
- 指定ページへ移動が使いやすいか
- Page Map クリックで目的ページへ移動できるか

### 4. 境界条件

- 1ページだけの PDF
- 先頭ページを1ページ目へ移動
- 末尾ページを最終ページへ移動
- 大量ページ PDF
- 複数 PDF 結合後のページ移動

## レビュー時の確認コマンド

```bash
git status
git diff
```

## 完了判断

`sprint/ACCEPTANCE_CHECKLIST.md` の全項目を確認する。
