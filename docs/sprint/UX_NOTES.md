# UX_NOTES

## Sprint 0.7.0 の UX 目標

ページ操作時に、ユーザーが「今どのページをどこへ動かしたか」を見失わないようにする。

## 現在の不便

### ドラッグ中に表示外へ移動できない

表示されている範囲内でしか移動できないと、大量ページ PDF ではページを遠くへ動かしにくい。

### 移動コマンドが細かすぎる

1ページずつの移動だけでは、10ページ以上動かすときに面倒。

### Page Map と実表示がズレると不安

左側の Page Map が古い状態のままだと、操作が成功したのか判断しづらい。

## 望ましい UX

- ドラッグ中に上下端で自然にスクロールする
- 遠くへ移動したいときは数値指定で移動できる
- 移動後も選択ページを見失わない
- Page Map にすぐ結果が反映される
- Status Bar に短く結果が表示される

## Page Map クリックの挙動

Page Map は「確認する場所」だけでなく「移動する場所」でもある。

クリックしたら、Page Viewer がそのページへ移動する。

## 移動後の視線誘導

移動後は、移動したページが見える位置にスクロールする。可能なら中央付近に表示する。

## Status Bar 文言例

- `12ページ目へ移動しました`
- `3ページ目を8ページ目へ移動しました`
- `1ページを削除しました`
- `Page Mapを更新しました`

## 避けたい UX

- 操作したのに見た目が変わらない
- Page Viewer と Page Map の順番が違う
- ドラッグ中のスクロールが速すぎる
- どのページが選択中かわからない
- エラー理由が表示されない
